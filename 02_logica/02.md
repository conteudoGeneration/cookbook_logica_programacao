<h1>Linguagem de Programação</h1>



Se o **Algoritmo** é o roteiro lógico e o **Raciocínio Lógico** é o pensamento organizado, a **Linguagem de Programação** é o meio que utilizamos para traduzir esse roteiro e pensamento para a máquina.

<br />

<h2>1. O Que É Uma Linguagem de Programação?</h2>



Antes de tudo, uma linguagem de programação é, de fato, uma **linguagem**. O objetivo principal de toda linguagem é **comunicar**.

Uma Linguagem de Programação é uma **ferramenta formal** que permite ao programador dar **instruções claras, detalhadas e sem ambiguidade** a um computador. Essas instruções permitem que a máquina realize ações específicas e alcance um objetivo predeterminado.

A chave é a **não ambiguidade**. Um programa deve funcionar da mesma maneira, entregando os mesmos resultados esperados, seja na sua máquina, na de um colega ou em um servidor distante. Para isso, o código precisa ser **preciso**, diferentemente da linguagem humana (português, inglês, entre outras.) que possui ambiguidade natural.

<div align="center"><img src="https://imgur.com/RGLuaZR.png" title="source: imgur.com" width="60%"/></div>

> **Programar nunca foi sobre código**; o código é apenas o **meio** pelo qual essa comunicação acontece no momento.

<br />

<h2>2. A Linguagem Humana vs. Linguagem da Máquina</h2>



O grande desafio da comunicação com máquinas está no nível de detalhe exigido.

| Característica  | Linguagem Humana                                            | Linguagem de Programação                                     |
| :-------------- | :---------------------------------------------------------- | :----------------------------------------------------------- |
| **Ambiguidade** | Alta. O ser humano preenche lacunas e interpreta intenções. | Baixa/Nula. Exige precisão e não permite interpretações.     |
| **Instrução**   | "Me encontre às 10h."                                       | `SE hora_atual = 10:00 E local = 'Rua X', ENTÃO exibir 'Mensagem de Encontro'.` |
| **Base**        | Vocabulário e contexto cultural.                            | Lógica, Matemática e Símbolos precisos.                      |

Para eliminar a ambiguidade comum da linguagem natural, as linguagens de programação utilizam **símbolos matemáticos** — como os **Operadores Relacionais e Lógicos** — para garantir que os significados sejam expressos de forma **exata e universal**.

<div align="center"><img src="https://imgur.com/ruCNnl4.png" title="source: imgur.com" /></div>

Na imagem acima, vemos os Operadores Lógicos e Relacionais, herdados da Matemática, em suas versões computacionais. Estes operadores são fundamentais, pois são eles que permitem ao computador realizar **comparações** e **tomar decisões**, transformando o raciocínio humano em instruções inequívocas para a máquina.

<h2>3. Linguagens de Alto Nível: A Ferramenta do Algoritmo</h2>



As linguagens que mais utilizamos hoje, como **Python, Java, JavaScript, entre outras**, são chamadas de **linguagens de alto nível**. Elas servem como a **ferramenta** para escrever e executar o raciocínio que foi criado no algoritmo. Na imagem abaixo, temos uma lista com as linguagens de programação de alto nível mais populares:

<div align="center"><img src="https://i.imgur.com/Gl697LP.png" title="source: imgur.com" /></div>

> Uma **Linguagem de Programação de Alto Nível** é um tipo de linguagem de programação que foi desenvolvida para ser **fácil de ler, escrever e entender pelos seres humanos**.

Embora a maioria dessas linguagens use termos específicos em **inglês** (`if`, `and`, `print`), o que realmente importa é a **lógica** por trás delas. Se você domina o algoritmo, você consegue escrever o programa em qualquer uma dessas ferramentas.

<br />

<h3>3.1. Legibilidade: A Comunicação Entre Humanos</h3>



Embora as linguagens de programação sejam escritas para máquinas, elas precisam ser compreendidas pelos **seres humanos**.

* A forma como a humanidade avança é por meio da **cooperação**.
* Você frequentemente trabalhará em equipe ou continuará um código escrito por outra pessoa.

Portanto, um bom código não é apenas aquele que funciona, mas aquele que também é **legível** e compreensível por outras pessoas desenvolvedoras.
